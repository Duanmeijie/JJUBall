# JJUBall 项目介绍说明书

## 一、项目概述

JJUBall 是面向九江学院乒乓球协会的微信小程序，解决协会活动报名、约球匹配、积分统计、赛季管理的数字化需求。项目采用微信云开发方案，实现零服务器、零域名、零备案的全栈开发。

### 1.1 核心定位

辅助工具，不替代微信群，降低管理成本，提升会员参与感。

### 1.2 核心价值

- 对会员：快速报名活动、查看积分排名、了解赛季进度
- 对管理员：高效管理活动、统计赛事数据、配置积分规则
- 对协会：数字化运营、数据可追溯、换届易交接

## 二、技术架构

### 2.1 技术栈

- 前端：微信小程序原生（WXML + WXSS + JavaScript ES6+）
- 后端：微信云开发（云函数 Node.js + 云数据库 + 云存储）
- 部署：微信云开发环境（无需独立服务器）
- 鉴权：微信原生登录（免鉴权获取 openid）

### 2.3 项目规模

- 前端页面：19 个

## 三、功能模块

### 3.1 用户系统

- 微信一键登录，自动注册
- 三级角色：普通会员（member）、裁判/干事（referee）、管理员（admin）
- 个人资料编辑（学号、姓名）
- 角色权限控制（前端拦截 + 后端校验）

### 3.2 活动管理

- 活动发布（仅管理员）：设置标题、类型、地点、时间、人数上限
- 活动列表：按状态、类型筛选，分页加载
- 活动报名：乐观锁防超卖，信誉分检查
- 取消报名：距开始时间 < 2小时扣信誉分
- 天气取消：不扣信誉分，批量通知

### 3.3 比赛积分系统

- 比赛录入：裁判/管理员选择对阵双方、输入比分
- 积分计算：使用数据库事务保证一致性
- 积分规则：可按赛事类型配置胜场/负场加分和权重系数
- 积分流水：每次变动有详细记录

### 3.4 赛季与段位

- 赛季管理：每学期一个赛季，可归档历史赛季
- 段位系统：青铜(0-99) / 白银(100-299) / 黄金(300-599) / 王者(600+)
- 赛季切换时当前积分清零，总积分保留

### 3.5 信誉分机制

- 初始 100 分
- 临时取消（<2小时）扣 10 分
- 无故缺席扣 20 分
- 天气取消免责
- 低于 60 分限制报名

### 3.6 消息系统

- 站内消息：活动通知、比赛结果、系统消息
- 订阅消息：报名成功、活动取消、比赛结果、段位晋升
- 未读消息红点提示

### 3.7 管理后台

- 数据概览（用户数、活动数、赛季信息）
- 用户管理（角色调整）
- 积分规则配置
- 信誉分规则配置
- 数据导出（CSV格式）
- 天气取消活动

## 四、数据库设计

共 11 个集合：

1. **users** - 用户表（openid、角色、积分、段位、信誉分）
2. **activities** - 活动表（标题、类型、地点、时间、人数、状态）
3. **activity_signups** - 报名表（活动ID、用户ID、状态）
4. **matches** - 比赛表（对阵双方、比分、胜者、赛季）
5. **score_rules** - 积分规则表（胜场/负场加分、权重）
6. **scores** - 积分流水表（变动记录）
7. **credit_rules** - 信誉分规则表
8. **credit_records** - 信誉分记录表
9. **seasons** - 赛季表（名称、起止日期、状态）
10. **messages** - 站内消息表
11. **feedbacks** - 反馈表

## 五、部署说明

### 5.1 环境要求

- 微信开发者工具 Stable 版
- 小程序 AppID：wxb1dbe132920d5c1c
- 最低基础库版本：2.14.0
- 需开通微信云开发

### 5.2 部署步骤

1. 微信开发者工具导入项目，填写 AppID
2. 在 app.js 中修改云开发环境 ID（env 字段）
3. 右键 cloudfunctions 目录，选择"创建并部署云开发环境"
4. 逐个右键云函数目录，选择"上传并部署：云端安装依赖"
5. 在云开发控制台创建 11 个数据库集合
6. 配置数据库权限规则
7. 在小程序后台配置订阅消息模板

### 5.3 环境 ID 配置

打开 miniprogram/app.js，将 `'your-env-id'` 替换为实际的云开发环境 ID。

- AppID 和环境 ID
- 源码仓库权限

### 6.2 新管理员上手步骤

1. 获取小程序后台管理员权限

3. 阅读本文档了解项目架构

5. 确认云函数正常运行
6. 尝试一次完整的活动发布和报名流程

### 6.3 日常维护

- 每学期初创建新赛季
- 根据需要调整积分规则
- 定期查看用户反馈并回复
- 每周导出一次数据备份
- 关注云开发控制台的调用量和存储用量
