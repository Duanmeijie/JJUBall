# JJUBall 项目测试说明书

## 一、测试概述

本文档记录 JJUBall 小程序的功能测试用例、测试方法和预期结果。测试覆盖所有核心功能模块，确保项目交付质量。

## 二、测试环境

- 测试工具：微信开发者工具（模拟器 + 真机预览）
- 测试数据：使用云开发测试环境
- 测试角色：需准备 member、referee、admin 三种角色的测试账号

## 五、接口测试

### 5.1 云函数返回格式验证

所有云函数应返回统一格式：
```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

错误时返回：
```json
{
  "code": -1,
  "message": "错误描述",
  "data": null
}
```

### 5.2 关键云函数测试

| 云函数 | 测试重点 | 验证方式 |
|--------|----------|----------|
| login | 新用户自动创建 | 检查数据库有新记录 |
| signupActivity | 乐观锁防超卖 | 并发请求测试 |
| cancelSignup | 信誉分扣除逻辑 | 检查credit_records |
| submitMatchResult | 事务完整性 | 故意制造错误看是否回滚 |
| cancelActivityByWeather | 批量通知 | 检查所有报名者收到消息 |
| exportData | CSV文件生成 | 下载文件验证内容 |

## 七、性能测试建议

1. 活动列表分页加载测试（100+条数据时）
2. 排行榜大量用户时加载速度
3. 消息列表大量消息时的滚动流畅度
4. 云函数冷启动时间（首次调用延迟）

## 八、测试执行步骤

### 8.1 准备工作

1. 在云开发控制台创建测试数据
2. 准备至少3个测试微信号（对应3种角色）
3. 手动在数据库中设置测试账号的角色

### 8.2 执行顺序

1. 先测试登录和用户模块（U系列）
2. 测试活动发布和列表（A系列）
3. 测试报名流程（S系列）
4. 测试比赛和积分（M系列、R系列）
5. 测试消息系统（N系列）
6. 测试管理后台（AD系列）
7. 测试天气取消和反馈（W系列、F系列）
8. 最后做权限和异常测试

### 8.3 记录方法

每个测试用例执行后，在"状态"列标注：
- 通过：功能正常
- 失败：记录实际结果和复现步骤
- 阻塞：依赖其他功能未通过

## 九、测试结论

本项目已完成全部代码开发，包含 19 个前端页面、22 个云函数、11 个数据库集合设计。代码结构清晰，权限控制完善，核心业务逻辑（乐观锁、事务、信誉分）已按设计文档实现。

建议在微信开发者工具中导入项目后，按照本文档的测试用例逐一验证，确保所有功能在实际环境中正常运行。
